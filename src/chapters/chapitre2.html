Les Tables et Relations

Dans ce chapitre, nous allons approfondir notre compréhension des bases de données en étudiant les tables et les relations qui existent entre elles. C'est un concept fondamental pour créer des bases de données efficaces et cohérentes.

Qu'est-ce qu'une table ?

Une table est une structure qui organise les données en lignes et en colonnes, un peu comme un tableau Excel. Chaque ligne représente un enregistrement, et chaque colonne représente un champ particulier.

Par exemple, dans une table "élèves", on pourrait avoir :

  • Colonne "id" : numéro d'identification unique
  • Colonne "nom" : nom de l'élève
  • Colonne "prénom" : prénom de l'élève
  • Colonne "classe" : la classe de l'élève

Les relations entre tables

Les bases de données relationnelles permettent de créer des liens entre différentes tables. Par exemple :

  • Table "élèves" avec un identifiant unique
  • Table "notes" qui contient les notes de chaque élève
  • La table "notes" contient une colonne "eleve_id" qui fait référence à l'identifiant de l'élève

Ce type de relation s'appelle une "clé étrangère".

Types de relations

Il existe différents types de relations entre les tables :

  • Un-à-un : Un élève a un casier
  • Un-à-plusieurs : Un professeur a plusieurs élèves
  • Plusieurs-à-plusieurs : Un élève suit plusieurs matières, une matière est suivie par plusieurs élèves
